Viya Travel

Eigene Logik für das Viya-Format. Getrennt von Emetravel und Exa Travel.

Fixe PMS-Codes: Double = DBL, Twin = TWI, Triple = TRI.
Pro Zimmer wird die Zeile 1.Room / 2.Room / 3.Room ... als Hauptzeile gelesen. Der Room Type wird aus der nächsten passenden Zeile mit Double / Twin / Triple / Single gelesen.

Eigene Logik für Exa Travel E-Mails. Nicht mit Viya oder Emetravel kombiniert.

Bey Tours

Eigene Logik für das Bey-Tours-Format. Getrennt von Viya, Emetravel und Exa Travel.

Die Datei wird über die Spalten PAX / APELLIDO PATERNO / APELLIDO MATERNO / PRIMER NOMBRE / SEGUNDO NOMBRE / TIPO DE HABITACIÓN gelesen.
In TIPO DE HABITACIÓN steht nur in der ersten Zeile eines Zimmers ein Wert (TPL, TWIN, DBL, SGL). Genau diese Zeile wird als Zimmer-Zeile verwendet.

Eurorutas

Eigene Logik für Eurorutas-PDF-Roominglisten. Getrennt von Viya, Emetravel, Exa Travel und Bey Tours.

Das Modul liest die Eurorutas PDF-Roomingliste mit Zeilen wie Nº / Name / Room.
Eine neue Zimmerzeile beginnt dort, wo ein Room-Code erkannt wird: DBL, TWN, TPL, SGL.
Folgende Zeilen ohne Room-Code werden demselben Zimmer zugeordnet. In / Out wird automatisch aus dem PDF gelesen.

Maseuropa

Eigene Logik für Maseuropa-PDF-Roominglisten. Die Zimmerart wird primär aus Observaciones gelesen.

Maseuropa-Regeln:
Bei 2 Adults wird die Zimmerart über Observaciones erkannt: DBL MAT, DOBLE MATRIMONIAL, DBL + INF, DBL+MNR, 1dbl-1bed, DOUBLE ROOM (1 bed) usw. ⇒ DBL.
TWIN, TWIN ROOM (2 beds), Oc 1 Habitacion Twin oder Text mit Twin/2 beds ⇒ TWI.
Bei 3 Adults wird immer TRI gesetzt. Bei 1 Adult wird für dieses Modul immer DBL exportiert.
